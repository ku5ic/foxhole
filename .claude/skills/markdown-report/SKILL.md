# Skill: Markdown report template

The markdown report is the primary human-readable output of Foxhole. It must read like it was written by a senior engineer, not generated by a linter. Tone is direct and professional. Language is plain. No jargon.

## Report section order

1. Title and metadata
2. Summary paragraph
3. Overall score
4. Category scores
5. Prioritized fixes
6. Findings by category
7. Performance metrics (if perf check was run)
8. Run metadata

## Title and metadata

```markdown
# Foxhole Audit Report

**URL:** https://example.com
**Audited:** 2024-03-15 14:32:01 UTC
**Duration:** 18.4s
```

## Summary paragraph

The summary is a single paragraph written in plain language. It states the overall condition of the frontend, names the two or three most significant issues, and ends with a clear statement about what to prioritize.

Rules:

- No bullet points in the summary.
- No WCAG clause references in the summary.
- No Lighthouse audit ID references.
- State the score and whether it passed the threshold if one was set.
- Name specific problems, not categories. "Six product images are missing alt text" not "accessibility issues were found".
- End with the single highest-priority action.

Example:

```
The page scored 74 out of 100, below the 80-point threshold. Six product images
are missing alternative text, which blocks screen reader users from understanding
the page content. LCP is 4.2 seconds on a simulated mobile connection, which will
affect search ranking. The most impactful fix is adding alt text to the product
images, which is low effort and resolves the most critical accessibility violation.
```

## Overall score

```markdown
## Score

**74 / 100** — Below threshold (80)
```

Or if passing:

```markdown
## Score

**86 / 100** — Pass
```

## Prioritized fixes

```markdown
## Prioritized fixes

### 1. Add alt text to product images

**Effort:** Low | **Severity:** Critical | **Category:** Accessibility

Six product images have no alternative text. Screen reader users receive no
information about these images. Add descriptive alt text to each image that
conveys its purpose in context.

Resolves: a11y-image-alt (6 instances)

---

### 2. Reduce render-blocking resources

**Effort:** Medium | **Severity:** Major | **Category:** Performance

Three CSS files and one JavaScript file are blocking the initial render.
Move non-critical CSS to load asynchronously and defer the JavaScript file.
This will improve FCP and reduce TBT.

Resolves: perf-render-blocking-resources

---
```

## Findings by category

Group findings by category. Within each category, sort by severity descending (critical first).

```markdown
## Findings

### Accessibility

#### [Critical] Images missing alternative text

**Selector:** `img.product-image`
**WCAG:** 1.1.1
**Recommendation:** Add descriptive alt text to each image. For decorative
images, use alt="".

---

#### [Major] Insufficient color contrast

**Selector:** `.price-label`
**WCAG:** 1.4.3
**Recommendation:** Increase the contrast ratio between the price label text
and its background to at least 4.5:1.

---
```

## Tone and language rules

- Sentence case everywhere. No title case headings.
- No em dashes. Use commas or rewrite the sentence.
- No smart quotes. Plain ASCII only.
- No passive voice in recommendations. "Add alt text" not "Alt text should be added".
- No hedging language. "This will improve FCP" not "This may potentially help with FCP".
- Numbers in findings are specific. "Six images" not "several images".
- Metric values include units. "4,200ms" not "4200".
- Bundle sizes in KB or MB with one decimal place. "842 KB" not "842000 bytes".
