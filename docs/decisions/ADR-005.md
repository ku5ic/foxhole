# ADR-005: Lighthouse uses a separate Chrome instance from Playwright

**Status:** Accepted, unification deferred to post-v1
**Date:** 2026-04-07

## Context

Foxhole uses Playwright to drive a Chromium browser for axe-core injection, semantic checks, and bundle analysis. It also uses Lighthouse for performance auditing. Both tools require a browser, but they manage their browser connections differently.

Playwright opens and controls a browser instance directly. Lighthouse opens its own Chrome instance via the `lighthouse` npm package's programmatic API. Connecting Lighthouse to an existing Playwright browser requires using Chrome DevTools Protocol (CDP) directly to pass the debugging port, which adds complexity and has historically been fragile across Lighthouse versions.

## Decision

For v1, allow Lighthouse to open and close its own Chrome instance separately from the Playwright browser. Each page audit that includes the `perf` check will use two browser processes: one Playwright Chromium for axe, semantic, and bundle checks, and one Chrome for Lighthouse.

This is documented as a known limitation in the README.

## Consequences

Each audit run against a URL with all checks enabled spawns two browser processes. On a machine with limited memory or CPU, this may cause one process to affect the performance measurements of the other. Lighthouse performance metrics are best captured in isolation; a concurrent Playwright browser may skew results slightly.

The practical impact is likely small for most users. Lighthouse already simulates throttled conditions and its scoring is relative to its own measurements, not wall-clock time.

Unification is deferred because it requires either: (a) using Playwright's CDP access to connect Lighthouse to the existing browser, which requires careful version alignment and testing, or (b) replacing the Lighthouse package with a custom implementation of the same audits, which is out of scope.

This will be revisited after v1.0.0 ships. The TODO comment in `src/runner/lighthouse.ts` marks the location where the integration point would change.
